## 1. 자문 의뢰 배경

ACTIGENCE는 한국 사회 문제 도메인(전세사기, 보이스피싱, 기획부동산 등)에서 **공익 데이터 분석 plugin**을 만드는 회사이며, 변호사가 소속되어 있지 않습니다. 본 회사가 만든 첫 plugin **jeonse-explorer**는 전세사기 피해자 또는 예비 임차인이 자신의 상황(보증금 구간·우선변제권 확보 여부·임대인 보유 주택 수 등 7개 객관 변수)을 입력하면, 사법연감·양형기준·국토교통부 통계·공개 표본 판례에서 비슷한 사건군의 **분포 통계**를 차트로 보여주는 도구입니다.

본 plugin은 **법률 자문이 아니며 사건 결과를 단정·예측하지 않습니다**. ACTIGENCE는 변호사를 매칭·추천하지 않고, 공공 지원 기관(주거안정 지원센터, 대한법률구조공단 등)만 안내합니다.

저희는 **변호사법 109조(법률사무 취급) 및 34조(공동수입)에 대한 회피 7원칙**을 코드 레벨에서 강제하는 컴플라이언스 안전 설계를 4개월간 진행했습니다:

1. 단정 결과 출력 금지 (분포 통계만)
2. 선택형(enum) 입력만 허용 (자유 텍스트 0개)
3. 공개 데이터만 사용 (유료 판례 DB 0건)
4. 변호사 매칭·추천 0개
5. "무료" 단어 0건
6. 판결문 양식 흉내 0건
7. 면책 배너·워터마크 100% 강제

본 7원칙은 `scripts/compliance-check.js`로 코드 자동 검증되며, GitHub Actions에서 매 push 시 PASS를 강제합니다 (C-01~C-10 검사). 또한 **PII 가드레일**이 가해자(임대인)·사용자 본인의 식별 정보(이름·전화·주민번호·계좌·주소·이메일)를 input에서 차단하도록 다중 레이어 정규식으로 설계되었습니다.

본 자문의 목적은 **위 설계가 변호사법 109조·34조 위반 위험을 충분히 회피하는지**, 그리고 **Anthropic Plugin Directory 정식 등재 시 추가로 보강해야 할 사항이 있는지**에 대한 객관적·전문적 검증을 받는 것입니다.

## 2. 자문 변호사 섭외 기준

| 우선순위 | 기준 |
|---|---|
| 1 | IT 법률 / 플랫폼 컴플라이언스 / AI 컴플라이언스 경험 |
| 2 | 변호사법 109조 (특히 1호 다목 — 전자적 방법) 관련 자문 또는 의견서 작성 경험 |
| 3 | 비변호사 운영 리걸테크 또는 데이터 분석 서비스에 대한 자문 경험 |
| 4 | 한국 IT 스타트업·SaaS·플랫폼 운영의 자율 결정 영역 자문 경험 |
| 5 | 변호사 협회 또는 사법부 출신으로서 협회 입장도 함께 검토 가능 |

**바람직한 후보 유형**:
- IT 전문 로펌의 컴플라이언스 그룹
- 리걸테크 자문 경험이 있는 단독 변호사
- AI/플랫폼 자문 경험이 있는 부티크 로펌

**바람직하지 않은 후보 유형**:
- 변호사 매칭 플랫폼의 이해관계자
- 송무 위주 사무실 (자문 영역에 대한 전문성 부족 가능)
- ACTIGENCE 운영 영역과 직접 경쟁관계인 회사의 자문 변호사

## 4. 자문 질문 (총 24개)

### B 단계 — 일반 컴플라이언스 검증 (17개)

> 본 plugin과 ACTIGENCE의 운영 방식이 변호사법 위반 위험에 노출되는지 1차 검증.

**1. 법률사무 취급 해당 여부 (변호사법 109조 1호 본문)**
본 plugin이 출력하는 "유사 사건군 X% 가 ~한 결과를 받았다"는 분포 통계 응답이 변호사법 109조 1호의 "법률사무 취급"에 해당하는지요? (저희는 사실 정보 제공이라고 보고 있으나, 검증 필요)

**2. 전자적 방법 — 109조 1호 다목**
2024년 개정으로 신설된 109조 1호 다목 ("전자적 방법으로 법률사무를 취급")이 본 plugin의 자동 분포 분석에 적용될 위험이 있는지요?

**3. 변호사법 34조 회피 — 공동수입**
plugin 사용에 대한 직접 과금 또는 광고 수익이 변호사법 34조(공동수입 분배 금지)에 저촉될 가능성이 있는지요? (현재는 직접 과금·광고 0건)

**4. ACTIGENCE의 비변호사 운영 정당성**
변호사가 소속되어 있지 않은 데이터 분석 회사가 본 plugin을 운영하는 것의 법적 정당성은 어떻게 확보됩니까? (광고·홍보 시 "변호사 미소속" 명시 의무 여부)

**5. 분포 통계 출력의 법률 자문 해당 여부**
사법연감·양형기준 등 공개 데이터에서 추출한 분포 통계 그 자체가 법률 자문에 해당할 가능성이 있는지요? (대법원 판례 기반)

**6. 사용자 자유 입력 차단 설계의 충분성**
본 plugin은 enum(선택형) 입력만 받고 자유 텍스트 입력을 100% 차단합니다. 이 설계가 변호사법 회피 근거로 충분한지요?

**7. 가해자 PII 처리 거부의 법적 의미**
가해자(임대인) 식별 정보를 input에서 정규식으로 차단하는 PII 가드레일의 법적 효력은? (가해자 명예훼손·개인정보보호법 측면)

**8. 워터마크·면책 배너의 법적 효력**
모든 응답에 표시되는 "본 정보는 법률 자문이 아닙니다" 워터마크 + 면책 배너가 사용자의 오해 책임을 ACTIGENCE에서 분리하는 데 충분한지요?

**9. 응답 단정 회피의 법적 근거**
"예상", "예측", "진단", "추천" 등 단정 단어 사용 금지 + "유사 사건군 X% 가 ~" 형태의 분포 표현만 사용하는 설계가 법률 자문 회피 근거로 충분한지요?

**10. 변호사 매칭·추천 0개의 의미**
변호사 이름·로펌·연락처를 일체 응답에 포함하지 않는 설계가 변호사법 34조 회피에 충분한지요? (공공 지원 기관만 안내함)

**11. 공공 지원 기관 안내만 허용의 정당성**
주거안정 지원센터, 대한법률구조공단 등 공공 기관만 안내하는 것이 합법적 안내의 범위 내에 있는지요?

**12. "법률 자문이 아닙니다" 명시의 법적 효력**
응답에 명시한 "본 plugin은 법률 자문이 아닙니다" 문구가 약관 또는 면책 동의로 인정될 가능성은?

**13. 사용자 데이터 stateless 정책의 GDPR/개인정보보호법 준수**
사용자 입력을 영속 저장하지 않고 분석 후 즉시 폐기하는 stateless 정책이 개인정보보호법·GDPR 준수에 충분한지요? (Supabase 인증·로그 layer만 보관)

**14. 향후 광고 게재 여부**
v1.0에는 광고 0건이지만, Phase 2에서 도메인 관련 공익 단체 광고를 게재할 계획이 있을 경우 변호사법 위반 위험이 발생하는지요?

**15. 수익 모델 합법성 (B2B SaaS)**
시민단체·법무법인·정부 부처 대상으로 plugin을 B2B SaaS 형태로 제공할 때 (예: 보고서 자동 생성 기능 추가), 변호사법 또는 변호사 단체 윤리규정에 저촉될 수 있는지요?

**16. 광고·홍보 시 표현 가이드라인**
ACTIGENCE가 본 plugin을 보도자료·블로그·SNS에서 소개할 때, 어떤 표현이 허용되고 어떤 표현이 위험한지요? (예: "전세사기 피해자를 위한 도구"는 OK, "변호사를 대체하는" 등은 NG)

**17. 변호사 협회 사전 협의 필요 여부**
본 plugin 출시 전 대한변호사협회 또는 서울지방변호사회와 사전 협의 또는 의견 청취가 필요한지요? (선례·관행 측면)

### C-1 단계 — 플랫폼·자동화 영역 추가 검증 (3개)

**18. Claude Code/Cowork plugin 호출 시 책임 소재**
사용자가 자연어로 trigger했을 때 plugin이 자동으로 호출되어 분석 결과를 반환하는 흐름에서, 만약 사용자가 그 결과를 법률 자문으로 오해해 손해를 입었다면 ACTIGENCE의 책임 범위는 어디까지인지요?

**19. Anthropic 검수 통과의 의미**
Anthropic Plugin Directory 검수를 통과해 등재되었다는 사실이 ACTIGENCE의 법적 면책 또는 컴플라이언스 정당성에 어느 정도 영향을 미치는지요? (해외 플랫폼의 검수가 한국 변호사법 적용에 미치는 효력)

**20. skill 자동 호출 vs 사용자 능동 호출의 법적 차이**
plugin이 사용자의 자연어를 자동으로 인식해 호출되는 (skill auto-trigger) 것과, 사용자가 명시적 명령(/jeonse-help)을 입력해 호출하는 것 사이의 법적 책임 차이가 있는지요?

### C-2 단계 — 핵심 검증 4개 ⭐ 필수 답변 요청

> Anthropic Plugin Directory 정식 등재 직전 마지막 컴플라이언스 검증 단계. 본 4개 질문에 대한 명확한 변호사 의견이 등재 신청 가능 여부를 좌우합니다.

**21. PDF 자동 추출 행위가 109조 1호 변형 해석 위험** ⭐
사용자가 임차계약서 PDF를 채팅에 던졌을 때 plugin이 자동으로 OCR·텍스트 추출하여 enum 값으로 매핑하는 행위가 변호사법 109조 1호의 "법률사무 취급"으로 변형 해석될 위험이 있는지요? (특히 PDF에서 가해자 정보를 자동으로 마스킹하는 동작이 추가될 경우)

**22. Cowork 자동화의 109조 1호 다목 적용** ⭐
Cowork 환경에서 plugin이 폴더 안의 여러 PDF를 일괄 처리하고 통합 보고서 PDF를 자동 생성하는 흐름이 109조 1호 다목("전자적 방법으로 법률사무를 취급")에 직접적으로 적용될 위험은? (B2B SaaS 모델로 시민단체·법무법인에 제공 시 위험 변화)

**23. 프롬프트 인젝션 우회 시 책임 범위** ⭐
사용자가 프롬프트 인젝션을 통해 회피 7원칙을 무력화하고 plugin이 단정 표현·변호사 추천 등을 출력하도록 만들었을 때, ACTIGENCE의 책임 범위는 어디까지인지요? (보안 가드레일을 코드 레벨에서 강제하지만 100% 차단은 어려운 한계 인정)

**24. Anthropic 검수 통과의 회피 근거 가능성** ⭐
Anthropic의 plugin 검수에서 "변호사법 109조·34조 회피 설계가 적절함" 판정을 받은 사실이, 향후 변호사 협회 또는 사법부의 사후 문제 제기 시 회피 근거 또는 신뢰 보호 원칙으로 작동할 가능성이 있는지요?
