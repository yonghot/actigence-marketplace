| 항목 | 값 |

| Marketplace 운영 주체 | ACTIGENCE (변호사 미소속 데이터 분석 회사) |
| 도메인 / 이메일 | actigence.ai / cio@actigence.ai |
| GitHub repo | https://github.com/yonghot/actigence-marketplace |
| Plugin v1.0.0 | jeonse-explorer (전세사기 분포 분석) |

| 모집 인원 | 5~10명 (선착순) |

## 1. 동행 커뮤니티 게시용 모집 메시지

> **제목**: 전세사기 분포 분석 plugin 베타 테스터 5~10명 모집합니다 (Claude Code 환경)
>
> 안녕하세요, ACTIGENCE 김용호입니다.
>
> 저희가 약 4개월간 작업해온 **Claude Code용 plugin**의 베타 테스트를 동행 커뮤니티에서 먼저 진행하고 싶어 글 남깁니다.
>
> ### 무엇을 만들었나요
>
> **jeonse-explorer** — 전세사기 피해자 또는 예비 임차인이 자신의 상황(보증금 구간, 우선변제권 확보 여부, 임대인 보유 주택 수 등)을 입력하면, **공개된 정부 통계와 표본 판례**에서 비슷한 사건군의 **분포 통계**를 차트로 보여주는 도구입니다.
>
> ### 무엇이 아닌가요 — 매우 중요
>
> 본 plugin은 **법률 자문이 아니며, 사건 결과를 단정·예측하지 않습니다**. 변호사를 매칭·추천하지도 않습니다. ACTIGENCE에는 변호사가 소속되어 있지 않은 데이터 분석 회사이고, **변호사법 109조·34조의 회피 설계**를 코드 레벨에서 강제했습니다 (회피 7원칙 — 단정 결과 출력 금지, 선택형 입력만, 공개 데이터만, 변호사 매칭 0건, 판결문 양식 흉내 0건, 면책 배너·워터마크 강제, 가해자 식별 정보 처리 거부).
>
> 정식 등재 전에 동행 분들의 솔직한 피드백을 받고 싶습니다.
>
> ### 참여 조건
>
> - Claude Pro 또는 Max 구독자
> - Claude Code CLI 또는 Cowork 환경 사용 경험 (스킬·MCP 서버·slash command 개념 익숙)
> - 본인 또는 지인이 전세 관련 의사결정을 한 경험이 있어 도구의 실용성을 평가할 수 있는 분

## 2. 베타 인스톨 가이드 (참여자 1:1 안내용)

신청자 확정 후 다음 가이드를 이메일 또는 DM으로 1:1 발송한다.

### 2.1 사전 조건

- Claude Code CLI 0.4.0 이상 또는 Cowork 데스크톱 앱
- Node.js 20 이상 (MCP 서버 빌드용 — 자동 처리되지만 설치되어 있어야 함)
- Git (marketplace clone용 — 자동 처리)

### 2.2 인스톨 명령

Claude Code 세션 안에서 다음 두 줄을 차례로 실행한다.

```
/plugin marketplace add yonghot/actigence-marketplace
/plugin install jeonse-explorer@actigence
```

설치가 완료되면 다음이 자동 활성화된다:

- 3개 skill (자연어 트리거 자동 발동): `jeonse-document-extractor`, `jeonse-fraud-analyzer`, `jeonse-law-summary`
- 1개 sub-agent: `compliance-guardrail` (응답 직전 회피 7원칙 검증)
- 1개 slash command: `/jeonse-help`
- MCP 서버 자동 시작 (stdio mode, 로컬 실행, 외부 네트워크 호출 0건)
- 6개 React UI 컴포넌트 (차트 + 면책 배너 + 워터마크)

### 2.3 첫 사용 — 3가지 테스트 시나리오

**시나리오 A — 가장 단순 (90초 내 결과)**

채팅창에 다음 자연어를 그대로 붙여넣는다 (자유 서술이 아니라 enum 매핑 안내가 자동으로 뜸):

> 보증금 1억 5천만원, 전입신고와 확정일자는 받았는데, 임대인이 100채 이상 가진 사람이래요. 경매 들어갔다고 하고, 수사 중이고, 다른 피해자 30명 정도 확인됐고, 서울이에요. 비슷한 사례가 어떻게 끝났는지 분포로 보여주세요.

**시나리오 B — 비교 시뮬레이션 (60초)**

> 보증금 1억과 2억 두 가지 가정으로 비교해서 회수율 분포 차이를 보여주세요. 나머지 조건은 동일.

**시나리오 C — 법령 일반 정보 (15초)**

> 전세사기특별법 적용 요건 5개를 표로 정리해주세요. 법률 자문은 필요 없고, 공식 출처 기반 일반 정보로요.

### 2.4 이런 동작이 나오면 정상

- ✅ 응답 상단에 **면책 배너** ("본 정보는 법률 자문이 아닙니다") 자동 표시
- ✅ 차트 위·아래에 **워터마크** ("ACTIGENCE / 분포 통계 / 법률 자문 아님")
- ✅ 입력은 **선택형(enum)** 으로만 받음 — 자유 서술 X
- ✅ 응답에 "예상", "예측", "진단", "추천" 등 단정 단어 0건
- ✅ "유사 사건군 X% 가 ~", "표본 N건 중 ~" 형태의 분포 표현
- ✅ 가해자(임대인) 휴대폰·이름·주소를 입력에 넣으면 즉시 거부 + 한국어 안내 메시지

### 2.5 이런 동작이 나오면 버그입니다 (꼭 알려주세요)

- ❌ "당신은 ~ 받게 될 것입니다" 류 단정 표현
- ❌ 변호사 이름·로펌·연락처 추천
- ❌ "무료" 단어 등장
- ❌ 면책 배너 누락
- ❌ 가해자 정보가 응답에 포함됨

발견 즉시 cio@actigence.ai 로 스크린샷과 함께 알려주세요. 즉시 hotfix 또는 v1.0.1 배포로 대응합니다.

## 3. 5문항 피드백 설문 (5점 척도)

베타 종료 시 참여자 전원에게 발송. Google Form 또는 이메일 회신.

각 문항은 5점 척도 (1=매우 그렇지 않다, 2=그렇지 않다, 3=보통, 4=그렇다, 5=매우 그렇다) + 1줄 코멘트.

| # | 문항 | 5점 척도 | 1줄 코멘트 |
|---|---|---|---|
| 1 | 자연어로 입력했을 때 plugin이 의도를 정확히 파악해 자동으로 작동했나요? (skill 트리거 정확도) | 1 / 2 / 3 / 4 / 5 | |
| 2 | 분포 차트가 한눈에 들어오고, 본인 사건 위치를 파악하기 쉬웠나요? (차트 가독성) | 1 / 2 / 3 / 4 / 5 | |
| 3 | 면책 배너·워터마크·"법률 자문 아님" 표시가 가시적이고, 도구의 한계를 명확히 신뢰할 수 있게 만들었나요? (컴플라이언스 신뢰도) | 1 / 2 / 3 / 4 / 5 | |
| 4 | 입력부터 결과 차트까지 90초 이내였나요? 속도가 의사결정 도구로 적절했나요? (응답 속도) | 1 / 2 / 3 / 4 / 5 | |
| 5 | 본인 업무·개인 의사결정·지인 도움 등 실제 상황에서 한 번이라도 더 쓸 것 같나요? (실용성) | 1 / 2 / 3 / 4 / 5 | |

## 5. 컴플라이언스 안내 (참여자에게 미리 공지)

- 본 plugin은 변호사법 109조·34조 회피 설계를 코드 레벨에서 강제함
- ACTIGENCE는 변호사가 소속되어 있지 않은 데이터 분석 회사
- 본 plugin은 법률 자문이 아니고, 변호사를 매칭·추천하지 않음
- 모든 응답은 공개 통계 기반 분포 정보이며 사건 결과를 단정하지 않음
- 가해자(임대인)·사용자 본인의 식별 정보(이름·전화·주민번호·계좌·주소)를 입력하면 즉시 차단되고 처리되지 않음
- 사용자 입력은 본 plugin 안에서 영속 저장되지 않음 (stateless)
- 분석에 사용된 데이터 출처: 사법연감, 양형위원회 양형기준, 국토교통부 통계, 공개된 표본 판례
