# Cowork 첫 경험 시연 가이드

본 문서는 ACTIGENCE Jeonse Explorer Plugin을 Claude Cowork 환경에서 시연하는 3가지 시나리오를 다룹니다. 동행 커뮤니티·언론 시연·CPN 파트너십 자료로 활용됩니다.

---

## 시연 핵심 가치 — "이게 진짜 미래다"

Cowork를 처음 접하는 사용자에게 90초 안에 다음을 체감시킵니다:

1. **자연어 한 마디**로 PDF가 자동 분석되는 경험
2. **시각적 결과**가 채팅에 인라인 렌더링되는 경험
3. **컴플라이언스 가드레일**이 가시적으로 동작하는 신뢰감
4. **공익적 임팩트**가 있는 첫 사용 경험
5. **확장 가능성**이 명확한 도구 (보이스피싱·투자사기로 확장 시연)

---

## 시연 환경 준비

### 사전 준비 (시연자)

- [ ] Claude Cowork 데스크톱 앱 설치 완료
- [ ] ACTIGENCE Jeonse Explorer Plugin 설치 완료
- [ ] 데스크톱 폴더에 시연용 샘플 파일 배치:
  - `Documents/Demo/임대차계약서_샘플.pdf`
  - `Documents/Demo/등기부등본_샘플.pdf`
  - `Documents/Demo/경매통지서_샘플.pdf`
  - `Documents/Demo/MultiCase/` (10건 PDF)
- [ ] 화면 녹화 도구 준비 (OBS, Quicktime 등)
- [ ] 시연용 가짜 데이터 사용 (실제 피해자 데이터 X)

### 샘플 파일 작성 가이드

샘플 PDF는 다음 조건을 만족해야 합니다:

- **가짜 식별 정보**: 임대인 "홍길동", 주소 "서울시 강남구 테헤란로 123" 등 명백히 가짜
- **객관적 변수는 다양**: 보증금·우선변제권·경매 단계 등은 실제 사례 분포 반영
- **PDF는 텍스트 기반**: 스캔 OCR 정확도 변수를 시연 시점에서 제거

## 시나리오 A — 단일 PDF 분석 (90초)

### 타깃 페르소나
- 동행 커뮤니티 멤버 (개발자·바이브코더)
- 첫 Cowork 사용자

### 시연 흐름

```
[0:00] 화면: Claude Cowork 메인 채팅창
       시연자: "내 데스크톱의 'Demo/임대차계약서_샘플.pdf' 분석해줘"

[0:05] Cowork: 파일 시스템 권한 요청 다이얼로그
       시연자: "허용" 클릭

[0:10] Plugin (jeonse-document-extractor Skill):
       "📄 'Demo/임대차계약서_샘플.pdf' 파일을 발견했습니다.
        분석을 시작하겠습니다."
       (스피너 애니메이션)

[0:25] Plugin: PDF OCR 완료 → 변수 추출 중

[0:30] Plugin (사용자 동의 단계):
       "📄 첨부 문서에서 다음 5개 변수를 추출했습니다:
        ✅ 보증금: 1억 5천만원
        ✅ 우선변제권: 전입신고 + 확정일자 모두 있음
        ⚠️ 임대인 보유주택 수: 추출 불가 (객관식 선택 필요)
        ✅ 경공매: 미진행 (현재 거주 중)
        ✅ 광역지역: 서울

        🛡️ 처리하지 않은 정보:
        - 임대인 식별 정보 (이름·주소·전화·사업자번호)
        - 사용자 본인 식별 정보 (이름·연락처)
        - 모든 입력값은 분석 후 즉시 폐기됩니다.

        2개 변수가 추출되지 않았습니다. 객관식으로 보충해주시겠어요?"

       객관식 표시:
       [ 임대인 보유주택 수: ① 1채 ② 2~10채 ③ 10~100채 ④ 100채+ ⑤ 모름 ]
       [ 임대인 수사 단계: ① 기소 ② 수사중 ③ 고소만 ④ 미고소 ⑤ 모름 ]

[0:45] 시연자: "② 2~10채" + "④ 미고소" 선택 → "진행" 클릭

[0:50] Plugin: MCP 도구 호출 → 응답 반환

[0:55] Plugin: Claude App UI 컴포넌트 인라인 렌더링
       - 면책 배너 (sticky, amber)
       - 도넛 차트: 특별법 적용 분포
       - 히스토그램: 회수 가능성 분포
       - 가로 막대: 양형 분포
       - 워터마크 (대각선, 반투명)
       - 공식 지원 기관 4곳

[1:25] Plugin (응답 마지막):
       "🛡️ 분석 완료. 모든 입력 데이터는 메모리에서 폐기되었습니다.
        ⚠️ 본 결과는 공개 판례·법령 통계 분석이며 법률 자문이 아닙니다."

[1:30] 시연자: 결과 보고 코멘트 → 시연 종료
```

### 시연 핵심 포인트 강조

시연 직후 다음 메시지를 강조:

1. **"PDF 한 번 던졌더니 90초 만에 결과가 나왔습니다."** (자연어 + 자동화)
2. **"가해자 정보는 처리하지 않습니다라고 명시되네요."** (가드레일 가시성)
3. **"차트가 채팅에 인라인으로 그려지네요."** (시각적 결과)
4. **"분석 후 즉시 폐기되었다고 알려주네요."** (Stateless 신뢰감)

## 시나리오 B — 비교 시뮬레이션 (60초)

### 타깃 페르소나
- 예비 임차인 (의사결정 보조)
- 부동산 투자 검토자

### 시연 흐름

```
[0:00] 시연자: "보증금 1억일 때와 2억일 때 사기 인정 비율 비교해줘"

[0:05] Plugin (jeonse-fraud-analyzer Skill):
       "두 시나리오를 비교 분석하겠습니다.
        나머지 변수는 어떻게 가정할까요?"

       객관식:
       [ 우선변제권: ① 둘 다 (전입+확정일자) ② 임차권등기 ③ 없음 ]
       [ 광역지역: ① 서울 ② 경기 ③ 인천 ④ 기타 ]

[0:15] 시연자: "① 둘 다" + "① 서울" 선택

[0:20] Plugin: MCP 도구 2회 병렬 호출 (deposit_range만 다름)

[0:30] Plugin: Claude App UI 비교 카드 렌더링
       - 좌측: "보증금 1억" 분포 3개 차트
       - 우측: "보증금 2억" 분포 3개 차트
       - 하단: 차이점 요약 (특별법 인정 -8%p, 회수율 중앙값 -12%p 등)

[0:55] Plugin: "⚠️ 본 결과는 비교용 통계 분포이며..."

[1:00] 시연자: "아, 보증금 한도 5억 근처에서 인정률이 크게 떨어지는구나"
```

### 시연 핵심 포인트

- **"한 번에 두 시나리오를 비교할 수 있다"** (병렬 추론)
- **"차이점을 자동 요약해준다"** (정형 출력)
- **"의사결정 도구로 활용 가능"** (실용 가치)

## 시나리오 C — 시민단체 폴더 일괄 분석 (120초)

### 타깃 페르소나
- 전세사기·깡통전세 피해자 전국대책위원회
- 시·도청 주거복지팀 실무자
- 정책 연구자

### 시연 흐름

```
[0:00] 시연자: "이 폴더 'Demo/MultiCase/'의 모든 사례 통합 보고서 만들어줘"

[0:05] Cowork: 폴더 권한 요청 → 시연자 "허용"

[0:10] Plugin: 10개 PDF 발견 → 일괄 OCR 시작 (병렬)
       (진행 표시: "10건 중 3건 처리 완료...")

[0:35] Plugin: 각 PDF에서 객관적 변수 추출 + 가해자 정보 마스킹
       이중 차단 작동 확인 메시지

[0:45] Plugin: "10건의 변수를 추출했습니다. 일괄 분석할까요?
                ⚠️ 이 과정에서 가해자 식별 정보는 모두 마스킹되었습니다."
       [ 일괄 진행 ]  [ 개별 검토 ]

[0:50] 시연자: "일괄 진행" 클릭

[0:55] Plugin: MCP 도구 10회 병렬 호출

[1:15] Plugin: 통합 분포 보고서 인라인 렌더링
       - 10건 평균 분포 + 개별 사건 표
       - 가해자 식별 정보 0건 (자동 검증 결과 표시)

[1:30] 시연자: "이거 PDF로 저장해줘"

[1:35] Cowork: 파일 저장 권한 요청 → 시연자 "허용"

[1:40] Plugin: 워터마크 포함 PDF 생성 → 데스크톱에 저장
       (사용자 데이터는 메모리에만 존재했음을 알림)

[2:00] 시연자: "10건 분석 + 보고서 PDF가 2분 만에 끝났네요"
```

### 시연 핵심 포인트

- **"수십 건 사례를 한 번에 처리"** (B2B 가치)
- **"보고서 PDF로 즉시 저장"** (실무 즉시 활용)
- **"가해자 정보 마스킹 자동 검증"** (컴플라이언스 신뢰)
- **"시민단체 SaaS 모델 가능성"** (확장)

## 시연 시 절대 피해야 할 것

### ❌ 하지 말 것

1. **실제 피해자 데이터 사용** — 반드시 가짜 샘플만
2. **단정적 멘트** — "이런 사건은 X로 결정됩니다" 같은 표현
3. **변호사 추천 요청** — "변호사 추천해달라고 해볼게요" 시연 X
4. **"무료 도구"라는 표현** — "별도 비용 없이 사용 가능" 사용
5. **법률 자문 도구라는 인상** — "데이터 분석 도구"로 일관 표현

### ✅ 해야 할 것

1. **"공익 데이터 분석 도구"** 일관 표현
2. **"공식 지원 기관에 문의하세요"** 멘트 강조
3. **가드레일이 보이는 순간** 멈춰서 강조 ("가해자 정보 처리하지 않습니다 표시 보이시죠?")
4. **분포·통계 표현만 사용** ("유사 사건군의 분포는...")
5. **ACTIGENCE 비변호사 회사 명시** ("저희 ACTIGENCE는 변호사가 소속되지 않은 데이터 분석 회사입니다")

## 베타 테스트 시연 후 피드백 수집

시연 직후 다음 5문항 피드백 (5점 척도):

1. 시연이 명확하고 이해하기 쉬웠나요?
2. 90초 안에 결과가 나오는 속도가 적절했나요?
3. 가드레일(컴플라이언스)이 가시적이고 신뢰가 갔나요?
4. 차트의 가독성이 좋았나요?
5. 본인이 직접 사용해보고 싶다는 생각이 드셨나요?

추가 자유 응답:
- 가장 인상적이었던 부분
- 개선이 필요한 부분
- 본인 업무에 적용 가능한 시나리오

피드백은 `cio@actigence.ai`로 수집하여 다음 패치에 반영.
