# Jeonse Document Extractor (PDF 자동 추출, Cowork 시연 핵심)

본 Skill은 Cowork 첫 경험 시연의 핵심이다. 사용자가 PDF 파일을 던지면 객관적 변수 7개만 자동 추출하여 분포 분석으로 연결한다. 가해자 정보는 절대 추출하지 않는다.

---

## 절대 규칙

1. **가해자(임대인) 식별 정보 절대 추출 금지**
   - 임대인 이름·주소·전화·사업자번호·주민번호 → 추출 결과에 포함 X
   - 응답에 다시 노출 X

2. **사용자(피해자) 식별 정보 추출 금지**
   - 본인 이름·연락처·주민번호 → 도구 input에 포함 X

3. **추출 가능한 변수는 7개로만 한정**
   - 보증금 액수
   - 우선변제권 (전입·확정일자·임차권등기·전세권)
   - 임대인 임차주택 보유 수
   - 경공매 진행 단계
   - 임대인 수사 단계
   - 다른 피해자 수
   - 거주 광역지역

4. **사용자 동의 강제**
   - 추출 직후 결과를 사용자에게 표시하고 "진행해도 될까요?" 한 번 확인
   - 사용자 "진행" 선택 후에만 도구 호출

5. **Stateless**
   - 추출 결과·도구 호출 결과 일체 파일 저장 X
   - 외부 전송 X
   - 세션 종료 시 자동 폐기

## 추출 절차

### Step 1: 파일 유형 분류

첨부 파일을 다음 카테고리로 분류:
- `rental_contract` (임대차 계약서)
- `property_register` (등기부등본)
- `auction_notice` (경매·공매 알림)
- `lease_registration_order` (임차권등기명령서)
- `judgment` (판결문)
- `news_article` (언론 기사)
- `other` (기타)

분류 불가 시 사용자에게 1회 확인 (객관식).

### Step 2: 객관적 변수 추출 (가해자 정보 강제 마스킹)

OCR 또는 텍스트 직접 추출 후 다음 매핑:

| 문서 종류 | 추출 가능 변수 | 추출 금지 항목 |
|---|---|---|
| 임대차 계약서 | 보증금 액수, 광역지역 | 임대인 이름·주소·전화, 임차인 이름·연락처 |
| 등기부등본 | 임대인 보유 주택 수 (제한적), 광역지역 | 임대인 이름·주민번호·주소 |
| 경공매 알림 | 경공매 단계, 광역지역 | 임대인·소유자 이름 |
| 임차권등기명령서 | 우선변제권 정보, 임차권등기 여부 | 모든 식별 정보 |
| 판결문 | 양형 결과 (참고용, 도구 input 비포함) | 피고인·피해자 이름 |
| 언론 기사 | 임대인 보유 주택 수, 다른 피해자 수, 수사 단계 | 임대인 이름 |

### Step 3: 사용자 확인 강제 표시

추출 직후 다음 형식으로 사용자에게 표시:

```
📄 첨부 문서에서 다음 객관적 변수만 추출했습니다:

✅ 보증금: 1억 5천만원
✅ 우선변제권: 전입신고 + 확정일자 모두 있음
⚠️ 임대인 보유주택 수: 추출 불가 (객관식 선택 필요)
✅ 경공매: 경매 진행 중
⚠️ 수사 단계: 추출 불가 (객관식 선택 필요)
✅ 다른 피해자: 본인만 (단일 사건)
✅ 광역지역: 서울

🛡️ 처리하지 않은 정보:
- 임대인 식별 정보 (이름·주소·전화·사업자번호)
- 사용자 본인 식별 정보 (이름·연락처)
- 모든 입력값은 분석 후 즉시 폐기됩니다.

추출 결과로 분석을 진행할까요?
[ 진행 ]  [ 추출 변수 수정 ]  [ 취소 ]
```

### Step 4: 보충 입력 (객관식 1회만)

추출 불가 변수가 있으면 사용자에게 객관식으로 한 번만 보충:

```
🔍 다음 2개 변수를 객관식으로 선택해주세요:

1. 임대인이 보유한 임차주택은 몇 채인가요?
   (1) 1채만  (2) 2~10채  (3) 10~100채  (4) 100채 이상  (5) 모름

2. 임대인 수사는 어느 단계인가요?
   (1) 기소됨  (2) 수사 중  (3) 고소 접수만  (4) 미고소  (5) 모름
```

자유 서술 답변 요청 절대 금지.

### Step 5: 사용자 "진행" 동의 후 도구 호출

`jeonse-fraud-analyzer` Skill로 위임 또는 직접 도구 호출. 단, 사용자가 "진행" 버튼을 누르지 않은 상태에서 도구 호출 절대 금지.

### Step 6: 결과 전달 + 폐기 알림

```
[Claude App UI 컴포넌트 자동 렌더링]

🛡️ 분석 완료. 모든 입력 데이터는 메모리에서 폐기되었습니다.

📌 공식 지원 기관:
- 국토교통부 전세사기피해자지원센터: jeonse.kgeop.go.kr
- 대한법률구조공단: 132
- 검찰청 범죄피해자 지원: spo.go.kr
- 대한변호사협회: koreanbar.or.kr

⚠️ 본 결과는 공개 판례·법령 통계 분석 결과이며, 법률 자문이 아닙니다.
```

## Cowork 시연 시나리오

### 시나리오 A — 단일 PDF 분석 (90초)

```
[0:00] 사용자: "내 데스크톱의 '임대차 계약서.pdf' 분석해줘"
[0:05] Cowork: 파일 시스템 권한 요청 → 사용자 승인
[0:10] Skill: PDF 발견 → OCR 시작
[0:25] Skill: 변수 추출 + 가해자 정보 마스킹
[0:30] Skill: "다음 5개 변수를 추출했습니다... 진행할까요?"
[0:45] 사용자: 객관식 2개 보충 + "진행" 클릭
[0:55] 도구 호출 → Claude App UI 렌더링
[1:30] 결과 + 폐기 알림
```

### 시나리오 B — 폴더 일괄 분석 (시민단체 시연, 120초)

```
[0:00] 사용자: "이 폴더의 모든 사례 통합 보고서 만들어줘" + 폴더 권한
[0:10] Skill: 10개 PDF 일괄 OCR (병렬)
[0:30] Skill: 각 사건의 가해자 정보 마스킹 (이중 차단)
[0:45] Skill: 10건 변수 추출 결과 표시 + 사용자 일괄 승인 요청
[1:00] 사용자: "진행"
[1:30] 도구 호출 10회 (병렬) → 통합 분포 차트
[2:00] PDF 저장 옵션 (워터마크 자동 삽입)
```

## 절대 금지 동작

- ❌ 가해자 이름·주소·전화 추출 또는 도구 input에 포함
- ❌ 사용자 본인 이름·연락처 추출 또는 도구 input에 포함
- ❌ 추출 결과를 파일로 저장 (Stateless 원칙)
- ❌ 외부 전송 (이메일·메시지 등)
- ❌ 사용자 "진행" 동의 없이 도구 호출
- ❌ 추출 결과로 단정적 답변 직접 생성 (반드시 도구 호출하여 분포로 답변)
- ❌ 저작권 보호 자료(판결문 본문) 100자 이상 그대로 인용
