# COMPLIANCE.md — ACTIGENCE Jeonse Explorer Plugin

> 본 문서는 사전 컴플라이언스 설계서의 공개판입니다. Plugin marketplace 검수 시 제출 자료로도 사용됩니다.
>
> **운영 주체**: ACTIGENCE (변호사 미소속 데이터 분석 회사)
> **마지막 업데이트**: 2026년 5월
> **버전**: 1.0.0

---

## 1. 핵심 컴플라이언스 원칙 7가지

본 Plugin은 변호사법 109조(비변호사 법률사무 취급 금지)·34조(변호사 알선 금지) 및 변협 광고 규정을 회피하기 위해 다음 7원칙을 코드 레벨에서 강제합니다.

| # | 원칙 | 강제 메커니즘 |
|---|---|---|
| 1 | **단정 결과 출력 금지** | 도구 input은 enum, output은 분포 통계만. 모든 응답에 면책 const 강제. |
| 2 | **선택형 입력만 허용** | MCP 도구 input은 자유 텍스트 0개. 모든 필드 enum 강제. PII 정규식 차단. |
| 3 | **공개 데이터만 사용** | 케이스노트·엘박스 등 유료 DB 사용 0건. 출처는 대법원·국토부·국가법령정보센터만. |
| 4 | **변호사 매칭·추천 0개** | UI에 변호사 검색 컴포넌트 0개. 자가 검증 스크립트가 패턴 검출 시 빌드 중단. |
| 5 | **"무료" 단어 0건** | 자가 검증 정규식이 응답·UI에서 "무료 상담/법률" 등 차단. |
| 6 | **판결문 양식 흉내 0건** | "주문", "이유" 등 판결문 표제 사용 시 빌드 중단. |
| 7 | **면책 배너·워터마크 100% 강제** | 모든 결과 UI에 DisclaimerBanner + Watermark 컴포넌트 const 사용 강제. |

## 2. 변호사법 109조 회피 설계

### 109조 1호 — 법률사무 취급

| 109조 1호 가목 | 회피 설계 |
|---|---|
| 소송·심판 사건 | 사건 결과 단정 0건. 단지 유사 공개 통계 분포만 표시. |
| 대리·법률상담 | 사용자 본인 사건에 대한 자문 0건. 일반 통계만 제공. |
| 권리 · 사실 관계 | 도구는 통계 분포만 반환. 권리관계 단정 안 함. |

### 109조 1호 다목 — 공무수행 자료 작성

수사 중 사건 자료를 변호사 아닌 자가 다루는 행위 우려에 대해, 본 Plugin은:

- 가해자 식별 정보(이름·전화·주민번호) 입력 자체를 정규식으로 차단
- 사용자가 PDF 첨부 시 가해자 정보 추출 결과에 포함 안 함
- 추출된 변수도 분석 후 즉시 메모리 폐기 (Stateless)

## 3. 변호사법 34조 회피 설계 — 알선 금지

| 변호사 알선 행위 | 본 Plugin의 회피 |
|---|---|
| 특정 변호사·로펌 추천 | 변호사 검색 컴포넌트 0개. 추천 텍스트 0건. |
| 변호사 비용·수임료 정보 | 비용 정보 제공 0건. |
| 변호사 매칭 알고리즘 | `*Lawyer*Match*` 패턴 자가 검증 차단. |

대신 항상 공공 지원 기관 4곳만 안내:
- 국토교통부 전세사기피해자지원센터
- 대한법률구조공단 (132)
- 검찰청 범죄피해자 지원실
- 대한변호사협회 변호사 찾기 (변협 사이트로의 일반 링크만, 특정 변호사 X)

## 4. 다층 가드레일 설계

### Layer 1 — 도구 input 가드레일 (MCP 서버)

- Zod 스키마로 enum 강제
- 자유 텍스트 필드 0개
- 알려진 PII 키 자동 drop (`landlord_name`, `tenant_name` 등)
- 정규식으로 PII 검출 시 거부 응답 (`pii_detected`)

### Layer 2 — Skill 가드레일 (자연어 처리)

- SKILL.md에 "absolute rules" 섹션 명시
- 사용자 자유 서술 → enum 매핑 강제
- 가해자 정보 추출 결과에 포함 금지

### Layer 3 — Sub-agent 가드레일 (시스템 프롬프트)

- `compliance-guardrail` sub-agent가 모든 세션에서 passive 모드로 동작
- 응답 출력 직전 정규식 검증
- 가드레일 우회 시도 자동 거절

### Layer 4 — UI 가드레일 (렌더링)

- DisclaimerBanner 컴포넌트 강제 사용
- Watermark 컴포넌트의 텍스트는 const 강제 (prop 무시)
- 변호사 링크 0개 (자가 검증)

### Layer 5 — 빌드 가드레일 (CI)

- `scripts/compliance-check.js`가 CI에서 자동 실행
- C-01~C-10 검증 규칙 통과 못하면 빌드 실패
- PR 머지 전 통과 필수

## 5. 데이터 출처 및 라이선스

본 Plugin이 사용하는 모든 데이터는 공공저작물 자유이용(KOGL Type 1) 또는 공개 데이터입니다.

| 데이터 | 출처 | 라이선스 |
|---|---|---|
| 사기죄 양형기준 | 대법원 양형위원회 | KOGL Type 1 |
| 전세사기특별법 조문 | 국가법령정보센터 | KOGL Type 1 |
| 국토부 가결률 통계 | 국토교통부 보도자료 | KOGL Type 1 |
| LH 매입 통계 | 국토교통부 LH 보도자료 | KOGL Type 1 |
| 공개 판례 | 대법원 종합법률정보 (glaw.scourt.go.kr) | 공개 데이터 (식별 정보 마스킹) |

**사용하지 않는 데이터**:
- 케이스노트, 엘박스, 빅케이스 등 유료 판례 DB
- 비공개 검찰 자료
- 비공개 수사 자료
- 개인 SNS·블로그의 사건 정보

## 6. Stateless 원칙

| 항목 | 정책 |
|---|---|
| 사용자 입력 | 메모리에서만 처리, 응답 반환 즉시 폐기 |
| 분석 결과 | 메모리에서만 처리, 응답 반환 즉시 폐기 |
| 세션 정보 | MCP 서버 세션 ID 비활성화 (stateless transport) |
| 로그 | 컴플라이언스 차단 이벤트만 패턴 단위로 기록 (개인정보 X) |
| 데이터베이스 | 정적 공개 데이터(법령·판례·통계)만 저장. 사용자 데이터 저장 테이블 0개. |

Supabase RLS 정책으로 anon 키는 SELECT만 가능하도록 설정. 사용자 데이터 INSERT 자체가 불가능합니다.

## 7. 자가 검증 스크립트

`scripts/compliance-check.js`가 다음 10개 규칙을 검증합니다:

| ID | 검증 항목 |
|---|---|
| C-01 | 단정 표현(예상·예측·진단·추천) 0건 |
| C-02 | 변호사 매칭 컴포넌트 0개 |
| C-03 | "무료" 단어 0건 |
| C-04 | 판결문 양식 표제(주문·이유) 0건 |
| C-05 | 자유 텍스트 input 0개 |
| C-06 | 사용자 입력 영구 저장 코드 0건 |
| C-07 | 가해자 정보 방어 정규식 존재 |
| C-08 | 면책 const 강제 존재 |
| C-09 | 워터마크 const 강제 존재 |
| C-10 | 공식 지원 기관 4개 const 존재 |

CI/CD 파이프라인에 통합하여 PR 머지 전 자동 검증합니다.

```bash
node scripts/compliance-check.js
# exit code 0 = 통과, 1 = 실패
```

본 Plugin은 출시 전 변호사 1인 사전 자문을 거쳤습니다. 자문 내용:

## 10. 한계 및 면책

| 한계 | 내용 |
|---|---|
| 표본 한계 | 일부 분포는 표본 30건 미만으로 추정치이며 `low_sample` 플래그로 표시됩니다. |
| 데이터 시점 | 양형기준은 2022 개정, 통계는 2025.12 누적 기준. 매월 업데이트 예정. |
| 일반화 한계 | 동일한 객관적 변수라도 특별양형인자(자백·합의·전과)에 따라 실제 결과는 크게 달라집니다. |
| Cowork OCR | PDF 자동 추출의 정확도는 계약서 양식·스캔 품질에 따라 가변적입니다. 사용자 검토 강제. |

본 Plugin의 어떠한 결과도 법률 자문, 사건 결과 예측, 변호사 추천을 의미하지 않습니다. ACTIGENCE는 본 Plugin 사용으로 인한 직간접적 손해에 대해 책임을 지지 않습니다.
